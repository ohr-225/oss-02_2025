# 오픈소스SW개론 과제 #2

## 1. top
### 개요
- 현재 시스템의 상태를 실시간으로 나타내는 명령어.
- 실행 중인 프로세스 목록, CPU 사용량, 메모리 사용량 등을 보여줌.


### 주요 옵션
| 옵션 | 설명 |
|------|------|
| -d | 업데이트 간격 변경 |
| -p | 특정 PID만 표시 |
| -u | 특정 사용자 기준으로 필터링 |
| -c | 명령줄 전체 표시 |
| -n | 반복 최대 수 지정 |
| -H | 스레드별로 표시 |


### 사용 예시
- `top`
  <img width="2290" height="408" alt="image" src="https://github.com/user-attachments/assets/86350869-1803-4131-b6d4-56b76cacea46" />

- -d : `top -d 1`
  - 1초마다 갱신함

- -p : `top -p 1234`
  - 특정한 PID (1234)만 표시함
 
- -u : `top -u chosun`
  - 특정한 사용자(chosun) 기준으로 필터링함

- -c : `top -c`
  - 명령어 대신 명령어 라인을 표시함
 
- -n : `top -n 5`
  - top이 화면을 5번만 갱신한 뒤에 자동으로 종료함
 
- -H : `top -H`
  - 각 프로세스의 스레드를 표시함

## 2. ps
### 개요
- 현재 실행 중인 프로세스 정보를 출력하는 명령어.
- PID, 사용자, 실행 중인 커맨드 등을 보여줌.


### 주요 옵션
| 옵션 | 설명 |
|------|------|
| -e | 모든 프로세스 출력 |
| -f | 풀 포맷 형태로 정보 출력 |
| -u | 사용자 프로세스 정보 출력 |
| -p | 특정 PID 지정해 출력 |


### 사용 예시
- `ps`
  <img width="1160" height="152" alt="image" src="https://github.com/user-attachments/assets/c628ee58-5446-49ce-aec7-84af0f1e4953" />

- -e : `ps -e`
  - 시스템 전체의 프로세스를 확인할 때 사용
 
- -f : `ps -f`
  - 부모 프로세스와 시간 등 자세히 표시
 
- -u : `ps -u chosun`
  - 특정 사용자(chosun)가 실행한 프로세스만 표시
 
- -p : `ps -p 1234`
  - 특정 프로세스(1234)의 상태만 단독으로 표시

## 3. jobs
### 개요
- 현재 셸에서 실행 중이거나 중지된 작업 목록을 보여줌.
- 실행 중인 프로그램이 어떤 상태인지 확인하는 명령어.


### 주요 옵션
| 옵션 | 설명 |
|------|------|
| -l | PID를 함께 출력 |
| -n | 마지막으로 상태가 변경된 작업만 출력 |
| -p | PID만 출력 |
| -r | 실행 중인 작업만 출력 |
| -s | 정지된 작업만 출력 |


### 사용 예시
- `jobs`
  <img width="1414" height="80" alt="image" src="https://github.com/user-attachments/assets/ff8f0441-b856-4002-a8d1-0d8575154d29" />

- -l : `jobs -l`
  - 원래 출력에 PID를 함께 출력

- -p : `jobs -p`
  - PID만 보고 싶을 때 사용

- -r : `jobs -r`
  - 현재 실행 중인 작업만 필터링해 출력

- -s : `jobs -s`
  - 현재 정지된 작업만 필터링해 출력

## 4. kill
### 개요
- 특정 프로세스에 시그널을 보내 종료하는 명령어.
- 기본적으로 종료를 위해 사용하며 다른 신호를 보낼 수도 있음.


### 주요 옵션
| 옵션 | 설명 |
|------|------|
| -l | 사용 가능한 모든 시그널 목록 출력 |
| -s | 지정한 시그널을 보냄 |
| -9 | 강제 종료 |
| -15 | 기본 종료 |
| -1 | 재시작 |
| -STOP | 프로세스를 정지함 |
| -CONT | 정지된 프로세스를 재개함 |


### 사용 예시
- `kill 1234`
  - 특정 PID(1234)에 정상 종료를 요청함

- -l : `kill -l`
  - 사용 가능한 모든 시그널 목록을 출력

- -s : `kill -s SIGSTOP 1234`
  - 특정 PID(1234)에게 지정한 시그널(SIGSTOP)을 보냄

- `kill -9 1234`
  - PID 1234의 프로세스를 강제로 종료

- `kill -15 1234`
  - PID 1234에 정상 종료 요청을 보냄

- `kill -1 1234`
  - PID 1234의 프로세스를 재시작함

- `kill -STOP 1234`
  - PID 1234를 정지 상태로 만듦

- `kill -CONT 1234`
  - PID 1234 (정지된 프로세스)를 재개함
